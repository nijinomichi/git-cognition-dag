# Git as a Distributed Temporal Cognition System

This repository explores Git not as a version control tool, but as a **distributed system of temporal cognition**.

We model Git as a Directed Acyclic Graph (DAG) that externalizes human memory, branching thought, and collaborative reasoning under uncertainty.

---

## Core Hypothesis

Git is a distributed graph-based memory system that externalizes human temporal cognition into a reproducible DAG, enabling error-tolerant collaboration.

---

## Formal Model

We define Git as:

G = (V, E, P, H)

Where:
- V: commits (state nodes)
- E: parent relationships (edges)
- P: pointers (branches, HEAD)
- H: history operations (merge, rebase, reset)

This structure forms a Directed Acyclic Graph (DAG) of evolving system states.

## Failure Taxonomy

We classify Git failures into:

- State misalignment (detached HEAD)
- History rewriting risks (force push)
- Merge semantic collapse (conflict ambiguity)

Each failure is interpreted as a structural property of distributed cognition.
